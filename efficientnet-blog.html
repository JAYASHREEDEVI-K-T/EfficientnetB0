01 — Introduction

Why This Project?

Transfer learning promises to let you borrow a model trained on millions of images and adapt it to your own task in a few hours on a single GPU. But the gap between "it runs" and "it generalizes well" is filled with subtle, silent traps.

In a previous attempt at this exact notebook, accuracy was near-random. The model structure was right. The optimizer was right. The bug was a single division — / 255.0 — applied before feeding images to a model whose first internal layer is already a normalization step. That one line quietly broke every prediction while the loss still appeared to converge normally.

This post documents everything that came after fixing that bug: three models, the actual epoch-by-epoch training output, and an honest breakdown of what each architectural and training change actually did.

02 — Dataset

The tf_flowers Dataset

We used TensorFlow's built-in tf_flowers: 3,670 images across five flower categories, split 70% train / 15% validation / 15% test. At batch size 32 that gives 81 training batches, 18 validation batches, and 18 test batches. Images vary in lighting, background, scale, and angle — a realistic diversity for a classification task.

03 — Preprocessing

The Preprocessing Pipeline — and the Silent Killer

Root Cause of the Previous Run's Failure

Dividing images by 255.0 before EfficientNetB0 shifts pixel values from [0, 255] to [0, 1]. EfficientNetB0's internal Rescaling layer is designed for raw [0, 255] float32 input — not pre-normalized input. Feeding pre-normalized values pushes activations far outside the distribution the pretrained weights expect, producing near-random outputs. The model still "trains" — loss decreases, the optimizer updates weights — but it's fitting garbage inputs to labels.

The fix is simple: cast to float32, resize, and stop. Let EfficientNet handle normalization internally. The sanity check at the end of the pipeline confirmed it worked: min=0.0, max=255.0.

# CORRECT — keep values in [0, 255], no division
def preprocess(image, label):
    image = tf.image.resize(image, [224, 224])
    image = tf.cast(image, tf.float32)  # ← no /255
    return image, label

# Augmentation pipeline — applied to training set only
data_augmentation = tf.keras.Sequential([
    RandomFlip('horizontal'),
    RandomRotation(0.2),
    RandomZoom(0.15),
    RandomContrast(0.2),
    RandomTranslation(0.1, 0.1),
    RandomBrightness(0.1),
])

Two separate pipelines were built: train_ds_base (no augmentation, used for the baseline model) and train_ds_aug (with augmentation, used for the regularized and fine-tuned models). Validation and test sets never receive augmentation.

05 — Baseline

Model 1 — Baseline: Demonstrating Overfitting

The baseline uses EfficientNetB0 with the top 20 layers unfrozen and a custom head of two bare Dense layers — no BatchNorm, no Dropout, no L2, no callbacks. It trained on train_ds_base (no augmentation) for 30 epochs with no early stopping. Its purpose is diagnostic: show what happens when a large unconstrained head is placed on a small dataset.

def build_baseline(num_classes):
    base = EfficientNetB0(include_top=False, weights='imagenet')
    base.trainable = True
    for layer in base.layers[:-20]:
        layer.trainable = False

    x = GlobalAveragePooling2D()(base.output)
    x = Dense(512, activation='relu')(x)  # no BatchNorm, no L2
    x = Dense(256, activation='relu')(x)  # no Dropout
    out = Dense(num_classes, activation='softmax')(x)
    # no EarlyStopping, no ReduceLROnPlateau

What This Gap Means

Training accuracy hit 99.88% while validation accuracy plateaued at 94.37%. That 5.51% gap is textbook overfitting. The two Dense layers — 655K+ combined parameters with no regularization — memorized training-set-specific patterns. Without augmentation, the model saw the same 2,500 images every epoch. Without EarlyStopping, it kept running 13 epochs past the point where validation improvement had stalled. The rising val_loss from epoch 6 onward is an early warning that went ignored because there was no callback watching for it.

06 — Regularized Model

Model 2 — Regularized: Closing the Gap to 1.88%

Same EfficientNetB0 base, same top-20 unfrozen layers, same head dimensions. The total trainable parameters are 4,843,176 — identical capacity to the baseline. What changed is every constraint governing how the model learns.

def build_regularized(num_classes):
    l2_reg = tf.keras.regularizers.l2(1e-4)

    x = GlobalAveragePooling2D()(base.output)
    x = BatchNormalization()(x)                               # new
    x = Dense(512, activation='relu', kernel_regularizer=l2_reg)(x)
    x = Dropout(0.5)(x)                                       # new
    x = Dense(256, activation='relu', kernel_regularizer=l2_reg)(x)
    x = Dropout(0.3)(x)                                       # new
    out = Dense(num_classes, activation='softmax')(x)

callbacks = [
    EarlyStopping(monitor='val_accuracy', patience=7,
                  restore_best_weights=True),
    ReduceLROnPlateau(monitor='val_loss', factor=0.5,
                      patience=3, min_lr=1e-7)
]

What Each Regularization Technique Does

Dropout 0.5 + 0.3

Dropout

Randomly zeros half the neurons in Dense-512 and 30% in Dense-256 each training step. No single neuron can memorize a training example. The network learns distributed, redundant representations instead of shortcuts.

L2 = 1e-4

L2 Regularization

Adds a penalty to the loss proportional to the square of each weight. Large weights — the mechanism of memorization — become expensive to maintain. The optimizer learns to prefer many small, broadly useful weights.

BatchNorm

Batch Normalization

Normalizes the 1,280 pooled features to zero mean and unit variance, then applies learned scale (γ) and shift (β). Stabilizes gradient flow through the head and adds mild noise that acts as an implicit regularizer.

6 augmentations

Data Augmentation

Random flip, rotation ±20°, zoom ±15%, contrast ±20%, translation ±10%, brightness ±10%. The model never sees the exact same image twice — memorizing individual pixel patterns becomes impossible.

EarlyStopping: Why "Restoring epoch 17" Matters

Training ran to epoch 24 before EarlyStopping triggered. But the best validation accuracy occurred at epoch 17. With restore_best_weights=True, the model was rewound to epoch 17's weights before being returned. Without this flag, you get the epoch 24 model — which is worse. This flag is not optional; it is the entire value of EarlyStopping for model quality.

The gap fell from 5.51% (baseline) to 1.88% — a reduction of two-thirds — using the same model capacity, simply by adding regularization constraints and better training hygiene.

07 — Fine-Tuning

Model 3 — Fine-Tuning: Unlocking Deeper EfficientNet Layers

Starting from the saved regularized model, we unfroze the top 50 layers of EfficientNetB0 (vs. top 20 previously) and recompiled at a much smaller learning rate.

# Unfreeze top 50 layers of the EfficientNetB0 base
eff_base = reg_model.layers[1]
eff_base.trainable = True
for layer in eff_base.layers[:-50]:
    layer.trainable = False

# Recompile with learning rate 20× smaller than regularized phase
reg_model.compile(
    optimizer=Adam(5e-6),   # was 1e-4
    loss='sparse_categorical_crossentropy',
    metrics=['accuracy']
)
Why 5e-6 and Not 1e-4?

The deep EfficientNetB0 layers encode features learned from 1.2 million ImageNet images. At 1e-4, gradient updates are large enough to overwrite those features — a phenomenon called catastrophic forgetting. At 5e-6, updates are tiny: weights shift gently toward the flower domain rather than being destroyed. The fine-tuning learning rate should generally be 10–50× smaller than the initial training learning rate.

09 — Lessons

Lessons Learned

1. Verify the input range before anything else

EfficientNetB0's internal Rescaling layer expects [0, 255] float32 input. Dividing by 255 beforehand was the root cause of a previous run's near-random outputs — and there was no obvious error message, only bad predictions. The single-line sanity check print(f"min={...}, max={...}") takes two seconds to write and would have caught it immediately. Add it to every pipeline.

2. High training accuracy is not the goal — the gap is

The baseline reached 99.88% training accuracy. That number is meaningless when val accuracy is 94.37%. The regularized model peaked at 96.61% training accuracy — lower — but its val accuracy was 95.10%. Reducing the gap from 5.51% to 1.88% with no architectural change is a more meaningful result than hitting a high training ceiling.

3. Regularization works as a system, not a toggle

Dropout, L2, BatchNorm, augmentation, EarlyStopping, and ReduceLROnPlateau each close a different pathway to overfitting. No single technique alone would have halved the gap. Dropout prevents neuron-level memorization. L2 prevents weight explosion. BatchNorm stabilizes gradient flow. Augmentation prevents pixel memorization. EarlyStopping prevents overtraining. ReduceLROnPlateau allows recovery near the current minimum. Together they form a system.

4. Fine-tune learning rate must be much smaller than initial LR

We used 5e-6 for fine-tuning vs. 1e-4 for the regularized phase — a 20× reduction. Larger updates during fine-tuning overwrite pretrained features rather than adapting them. The ratio between fine-tuning LR and initial training LR should generally be 10–50×.

5. EarlyStopping's best epoch is never the last epoch

In the regularized run, epoch 17 was the best. Training continued to epoch 24 before stopping. In the fine-tuning run, epoch 1 was the best. Training continued to epoch 6. Without restore_best_weights=True, both returned models would have been worse. This flag is not optional.

Recommended Sequence for Transfer Learning Projects

Fix and verify the preprocessing pipeline first with a pixel range sanity check. Train with a frozen base and confirm correct output shapes. Add all regularization techniques as a group — not one at a time. Only fine-tune after the head is stable. Keep the fine-tuning learning rate at least 10× below the initial LR. This sequence prevents chasing bugs caused by a broken pipeline underneath everything else.
